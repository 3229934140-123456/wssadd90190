# 监管处罚应对命令行检索工具 (Penalty Retrieval CLI)

供法务助理、实习生和外聘律师在整理大量处罚资料时快速查询内部口径的命令行工具。操作极简，仅包含 **导入**、**搜索**、**导出** 三类核心命令。

---

## 功能特点

- 📁 **批量导入**：将历年处罚决定书、申辩意见、整改报告放入文件夹，一键导入
- 🔍 **智能搜索**：支持多关键词、多维度过滤（公司/监管/业务线/标签）
- 📄 **脱敏导出**：导出简版参考清单，自动隐藏敏感联系人和内部金额测算
- 💡 **交互友好**：ASCII 图形化界面，操作提示清晰
- 🗄️ **零配置**：基于 SQLite，无需安装数据库服务

## 快速开始

### 环境要求

- Python 3.8+
- 无需额外依赖（纯 txt/md 文档即可运行）
- 可选依赖（读取 docx/pdf 时需要）：
  ```bash
  pip install -r requirements.txt
  ```

### 启动工具

**交互式模式（推荐）：**
```bash
python main.py
```

启动后会进入交互式命令行，输入 `help` 查看全部命令。

**直接命令模式（脚本调用）：**
```bash
# 搜索案例
python main.py search "广告绝对化用语"

# 导入文档
python main.py import ./documents/2024_cases

# 查看统计
python main.py stats
```

## 核心命令

### 1. 导入 `import`

将企业历年处罚资料放入指定文件夹（支持 `.txt` `.md` `.docx` `.pdf`），然后：

```
[penalty]> import ./处罚资料/2023年广告类
```

按提示录入：
- **所属公司**（必填）：如 "XX科技有限公司"
- **监管部门**（必填）：如 "市监局"、"网信办"、"人社局"
- **业务线**（可选）：如 "广告业务"、"人力资源"、"数据合规"
- **案件标签**（可选）：多个用逗号分隔，如 `广告法,绝对化用语,虚假宣传`
- **处罚日期**（可选）：格式 `YYYY-MM-DD`

> 💡 工具会自动合并关联文档（如同一案件的处罚决定书+申辩意见+整改报告），并自动提取罚款金额和生成摘要。

### 2. 搜索 `search`

```
[penalty]> search <关键词1> [关键词2] ... [选项]

选项：
  -c <公司>       按所属公司过滤
  -r <监管部门>    按监管部门过滤
  -b <业务线>     按业务线过滤
  -t <标签>       按案件标签过滤
  -n <数量>       返回结果数量上限（默认50）
```

**搜索示例：**

```bash
# 单关键词
search 广告绝对化用语

# 多关键词（AND关系）
search 个人信息 告知同意

# 带过滤条件
search 未及时签劳动合同 -b 人力资源 -n 30

# 按公司+监管过滤
search 虚假宣传 -c XX科技 -r 市监局
```

搜索结果会以列表形式展示，包括：
- 案例编号（如 `TECH-2024-0001`）
- 处罚金额
- 处理结果摘要
- 所属公司、监管部门、业务线、标签

### 3. 查看详情 `view`

```
[penalty]> view <编号>
[penalty]> view <案例号>
```

示例：
```bash
view 1              # 查看上次搜索结果中第1条
view TECH-2024-0001 # 按案例号查看
```

详情包含：违法事实、处理结果、申辩意见、整改报告、可借鉴话术。

### 4. 导出 `export`

```
[penalty]> export <编号列表> [-f 格式] [-o 路径] [-t 标题]

格式：md（默认）、txt、json
编号格式：1   1,3,5   2-6   CASE-001,CASE-002
```

**导出示例：**

```bash
# 导出第1、3、5条为Markdown
export 1,3,5

# 导出第2-4条为TXT到指定路径
export 2-4 -f txt -o ./给律师的参考清单.txt

# 按案例号导出JSON
export TECH-2024-0001 -f json

# 自定义标题
export 1,2 -t "广告类处罚参考-供律师审阅"
```

**导出清单内容（简版）：**
- ✅ 保留：基本信息、违法事实、处理结果、申辩要点、整改措施、可借鉴话术
- ❌ 隐藏：敏感联系人、内部金额测算
- 🔒 自动脱敏：手机号、邮箱、身份证号等

### 5. 其他命令

| 命令 | 说明 |
|------|------|
| `list [-n 数量]` | 列出所有案例 |
| `stats` | 查看数据库统计（案例数/总金额/公司/监管/标签） |
| `delete <编号>` | 删除指定案例（需二次确认） |
| `help <命令>` | 查看命令详细帮助 |
| `quit` / `exit` | 退出工具 |

## 典型使用场景

### 场景一：整理全年处罚资料

1. 从企业内网下载全年处罚决定书、申辩意见、整改报告
2. 按案件归档到文件夹（如 `2024处罚/市监局广告类/`）
3. 执行 `import` 命令，录入元数据
4. 用 `stats` 检查导入结果

### 场景二：律师来函询问某类案件口径

1. 启动工具，执行 `search 广告绝对化用语`
2. 浏览结果，`view 2` 查看典型案例详情
3. 选择3-5个代表性案例：`export 1,3,5 -t "广告用语处罚参考-供XX律师审阅" -o ./发送给律师/`
4. 将生成的 Markdown 文件直接发送

### 场景三：新人培训

1. 新人问："我们之前遇到过个人信息合规的处罚吗？"
2. `search 个人信息 告知同意 -b 数据合规`
3. 直接展示搜索结果列表，快速了解历史处理口径

## 数据安全提示

- 数据库文件 `data/penalty.db` 包含完整内部信息，请妥善保管，不要提交到 Git
- 导出的简版清单已脱敏，但发送前建议人工复核
- 内部金额测算和敏感联系人字段**不会**被导出

## 常见问题

**Q: 支持哪些文档格式？**
A: 原生支持 `.txt` 和 `.md`。安装 `python-docx` 后支持 `.docx`，安装 `pdfplumber` 后支持 `.pdf`。

**Q: 导入时会自动提取哪些信息？**
A: 罚款金额（从文本中匹配正则）、案件摘要（自动截取文档开头）、文档类型分类（处罚决定书/申辩/整改）。

**Q: 数据库在哪里？可以迁移吗？**
A: 默认在 `data/penalty.db`，是标准 SQLite 文件，可直接复制备份或迁移。

**Q: 案例编号规则是什么？**
A: `公司缩写-年份-序号`，例如 `TECH-2024-0001`，可在导入时自定义。
